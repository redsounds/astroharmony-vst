# AstroHarmony — UI Specification v1

## Design Source
- **Reference app:** AstroHarmony Electron desktop app at `D:\Claude\Sessions\cinematic-composer` (READ-ONLY, never modify)
- **Reference component:** `components/v2/WriteWorkspaceV2.tsx` — the canonical layout root
- **Reference theme:** `app/globals.css` — `:root --cc-*` token block (cc-dark theme)
- **Reference screenshots:** the FL Studio test screenshot we annotated together (`Trumpet` instrument, `Epic` mood, 195 BPM, 4-chord progression `Cadd9 → Em → F△7 → Gsus4`)

## ⚠ Document-vs-implementation note
This spec describes the **target visual + structural layout**. Unlike SimpleGain — where `v1-test.html` *became* `Source/ui/public/index.html` — AstroHarmony's production HTML is **generated by Vite from React components** during Sub-phase B of `/impl`. So:

- `v1-test.html` is a **static wireframe** for design approval only — it shows the layout grid, color theme, and section sizes filled with placeholder content
- The real UI comes from porting `components/v2/*.tsx` (14 components, ~2,200 lines) and bundling via `vite build` → single-file HTML → embedded as JUCE BinaryData
- The visual fidelity target is "indistinguishable from the Electron app at the same window size"

## Window

### Default size
**1280 × 800 px** — chosen to fit the full layout at slightly compressed right-panel width while staying inside the comfortable bounds of plugin windows in major DAWs (FL Studio, Reaper, Ableton).

Math:
- Left mood sidebar: **230 px** (matches `WriteWorkspaceV2.tsx` constant)
- Main center: ~**595 px** (chord grid + suggestions + scale picker fit comfortably)
- Drag divider: **5 px**
- Right panel: **450 px** default (override of source's 580 to fit plugin window; user can drag wider)
- Top bar: **56 px**
- Bottom bar: **48 px**
- Total: 230 + 595 + 5 + 450 = **1280**, 56 + 696 + 48 = **800**

### Resizable
**Yes.** JUCE plugin window resize enabled.
- Min: **960 × 700** (still usable)
- Max: **1920 × 1200** (full HD)
- Right panel internal resize (drag divider) clamped to **290–820 px** as in source

### DPI scaling
Honor host DPI via JUCE's `getDesktopScaleFactor()`. WebView CSS uses `rem` units so OS-level scaling cascades naturally.

## Component Inventory

All ported from `components/v2/` — listed with role + source line counts.

### 1. TopBar (`TopBar.tsx`, 279 lines)
**Position:** Grid area `top`. Full-width, 56 px tall.

**Left cluster:**
- Logo mark (cyan diamond) + "AstroHarmony" wordmark
- Projects ▾ dropdown (opens ProjectMenu popover)
- Save icon
- Undo / Redo buttons (disabled when no history)

**Center:**
- "ASTROHARMONY" centered uppercase wordmark in `--cc-accent-l` color (Fraunces serif tracking-wide)

**Right cluster:**
- InstrumentPickerTop (Trumpet 🎺 / Piano / Flute / Strings ▾)
- Play ▶ button
- Stop ⏹ button
- Loop 🔄 button (toggles `loop` APVTS param)
- Tempo display `195 BPM ▾` (clicking opens BPM input)
- Volume slider `🔊 ━━●━━ 80%`
- Settings ⚙ icon

### 2. MoodSidebar (`MoodSidebar.tsx`, 104 lines)
**Position:** Grid area `left`. 230 px wide.

**Header label:** "MOOD" (small caps, muted)

**10 mood buttons** stacked vertically, each:
- Emoji (Epic 👑 / Dark 🌑 / Hopeful 🌟 / Sad 💧 / Tension ⚡ / Mystery ❓ / Fantasy 🧚 / Adventure 🗺 / Love 💗 / Horror 👻)
- Title in serif (Fraunces 14px)
- One-line description in sans (DM Sans 11px muted)
- Active = elevated card surface with accent-tinted left border (4px purple)
- Hover = lighter surface

### 3. ProgressionV2 (`ProgressionV2.tsx`, 464 lines)
**Position:** Top of grid area `centre`.

**Header row:**
- Project title (editable, click-to-rename) + bars-total display (e.g., "4 bars total")
- CLEAR button (right-aligned, ghost style)

**ChordCard grid (`ChordCardV2.tsx`, 249 lines):**
- DnD-enabled horizontal layout (cards flow in row; wraps if many)
- Each card: ~190 × 110 px, rounded 10 px, tinted background based on `cc-card-N` palette
- Transition annotations between cards ("1 to 2", "2 to 3"...)
- Per-card contents:
  - Top-left: Roman numeral (small muted)
  - Center: chord name (Fraunces, 1.7 rem)
  - Bottom-left: scale tag (e.g., "C IONIAN" small uppercase)
  - Bottom-right: bar count badge ("1bar")
- Hover reveals: ✕ delete (top-right), ⠿ drag handle (top-left)
- Active state during playback: accent ring + glow

**Progress bar:** Full-width 4 px gradient (cyan → pink) under the cards, fills based on `currentBeatChanged` event

### 4. SuggestedChords (`SuggestedChords.tsx`, 145 lines)
**Position:** Below ProgressionV2.

**Header:** "SUGGESTED NEXT CHORDS" small caps + active-mood badge (e.g., "EPIC ・ 6")

**6 suggestion cards** in a row:
- Top label: short hand label ("Triumphant Open", "Breeding Anchor", etc. — mood-specific from `lib/moodEngine/data/*.ts`)
- Center: chord name (Fraunces, 1.4 rem)
- Bottom: roman numeral
- Star/spark indicator (cyan) showing it's a suggestion
- ＋ add button (right) — appends to progression

### 5. DarkChordPicker (`DarkChordPicker.tsx`, 45 lines) + ScaleExtensionRow (`ScaleExtensionRow.tsx`, 175 lines)
**Position:** Bottom of grid area `centre`.

**Scale/Mode header:**
- "SCALE / MODE" label
- Root note ▾ select (12 chromatic, sharps as ♯)
- Mode ▾ select (★ Ionian (Major), Mixolydian, Aeolian — full 21 modes available)
- Extension toggle buttons: Triad | 6&7 | 9 | 11 | 13 (multi-select, active = cyan)
- Mood preferences row: "Epic prefers: Ionian (Major) | Mixolydian | Aeolian" (clickable to swap mode)

**Column headers:** `I  II  III  IV  V  VI  VII` (7-column grid)

**Chord rows (one ScaleExtensionRow per selected extension):**
For each extension chosen, a row with section label and 7 chord buttons + ＋:
- "Triads" row: C, Dm, Em, F, G, Am, Bdim
- "13th chords" row: C△13, Dm13, Em7(♭9,11,♭13), F△13(#11), G13, Am11(♭13), Bø7(...)
- "sus2", "sus2+13", "sus4", "sus4+13" rows
- Non-standard sus cells show special labels (Ephry, Flyd) in `--cc-accent-l`
- Empty cells are present but transparent (grid alignment)

### 6. RightPanel (`RightPanel.tsx`, 403 lines)
**Position:** Grid area `right`. Default 450 px, drag-resizable 290–820.

**Sections (top to bottom):**

## Control → Parameter ID Mapping

Per [parameter-spec.md](../.ideas/parameter-spec.md) for IDs. JS event ↔ DOM attribute mapping for the bridge:

| UI control | Triggers | Reads from |
|---|---|---|
| Play ▶ button | `playProgression(currentProgression, loop)` | `playStateChanged` event → updates icon to ❚❚ |
| Stop ⏹ button | `stopAll()` | n/a |
| Loop 🔄 button | sets `loop` APVTS param via WebToggleButtonRelay | APVTS round-trip |
| Tempo display | sets `tempo` APVTS param via WebSliderRelay (or value input) | `hostBpmChanged` event when host-synced |
| Volume slider | sets `master_volume` APVTS param via WebSliderRelay | APVTS round-trip |
| Pitch slider (bottom) | sets `pitch_transpose` APVTS param via WebSliderRelay | APVTS round-trip |
| Instrument picker | calls `setInstrument(id)` native function | Zustand `activeInstrument` |
| Mood sidebar | mutates Zustand `activeMood` → `pushState` | hydrated from C++ on load |
| Chord clicks (any) | calls `playChord(notes, voicing)` + mutates Zustand `progression` | `previewNotes` event |
| Projects dropdown | calls `listSessions`, `loadSession`, `saveCurrentSession`, etc. | n/a |
| EXPORT MIDI | builds MIDI bytes via `lib/midi.ts` → `exportMidi(bytes)` | result toast |
| ChordCard delete | mutates Zustand → `pushState` | n/a |
| ChordCard drag-drop | mutates Zustand → `pushState` (committed on drop only) | n/a |

## Animations

- ChordCard mount: `fade-in 200ms ease-out` (`@keyframes` from globals.css)
- Active ChordCard during playback: subtle 1px ring glow pulse (`box-shadow` animation, ~600ms)
- Hover state transitions: 120ms ease
- Suggestion cards refresh: stagger fade-in (~60ms per card)
- Mood switch: cross-fade suggestions panel (~250ms)

## Accessibility

- Tab order: TopBar left → center → right → MoodSidebar → main content top→bottom → RightPanel → BottomBar
- Focus ring: 2 px `--cc-accent` outline, 2 px offset
- All buttons have `aria-label`
- Color is never the sole indicator (active states also have icon/text change)

## Out-of-Scope (for v1 design)

- Color theme switching (light/cream theme is in globals.css `@theme` block; not exposed in plugin v1)
- Multiple workspace tabs
- Tooltips (defer to v1.1 — overlay positioning inside a plugin window has edge cases)
- Mobile/touch layouts (irrelevant for VST)

## DOM ID Conventions for Bridge

For elements that the C++ side may need to address directly (rare — mostly for `previewNotes` highlighting):

```
.app-root            — top-level grid container
[data-pane="top"]    — TopBar
[data-pane="left"]   — MoodSidebar
[data-pane="centre"] — Main scrollable area
[data-pane="right"]  — RightPanel
[data-pane="bottom"] — BottomBar
[data-mood="<id>"]   — individual mood buttons (id ∈ moodEngine list)
[data-chord-id]      — ChordCards in progression
.piano-key[data-note="<note>"] — mini-piano keys (for highlight)
```

This is intentionally minimal — most state flows via Zustand, not direct DOM manipulation.
